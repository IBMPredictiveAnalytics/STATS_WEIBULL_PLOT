STATS WEIBULL PLOT Extension Command

Calculate and display a three-parameter Weibull plot

STATS WEIBULL PLOT
TIME = variable*
COUNT = variable
TYPE = variable
FAILURE=“failure string”
SUSPENSION = “suspension string”
ITEMNAME = variable

/OPTIONS
TITLE = “chart title”
ANNOTATION = “chart annotation”

/SAVE
GRAPHDS = dataset
GRAPHFILE = "file specification"
TEXTFILE = "file specification"

FILEMODE = OVERWRITE** or APPEND

/HELP

* Required
** Default

STATS WEIBULL PLOT /HELP prints this information and does nothing else.

STATS WEIBULL PLOT TIME=time TYPE=FailOrSusp COUNT=number.

TIME specifies the time of the observation from 0. This is not an SPSS time variable. It could be measured in time units such as weeks, but it could also be measured in terms of a measure such as number of usage cycles. The values must be positive.

COUNT specifies the count for each time period. If no COUNT variable is specified, each case counts as one.

TYPE specifies whether the case represents a failure or a suspension, i.e. right-censored observation. If it is omitted, all cases are considered to be failures.

FAILURE and SUSPENSION specify the codes for the TYPE variable that define the type ofthe case. They default to “F” and “S”, respectively. Cases that do not match either of these codes are discarded from the analysis.

ITEMNAME can specifiy a variable containing the name of the item being processed. It will be used for labelling the output.

OPTIONS

TITLE specifies a title for the chart. If the text contains )ID and there is an itemname, the itemname will replace that text.

ANNOTATION specifies an annotation that appears on the X axis. The date and time are automatically appended.

SAVE

GRAPHDS specifies a dataset name. If supplied, the dataset containing the points plotted in the chart is retained under that name.

GRAPHFILE specifies a file name (include the extension). If supplied, an sav file containing the points plotted in the chart is written. The contents are the same as for GRAPHDS. The choice is a matter of convenience.

TEXTFILE specifies a text file to which the points plotted in the chart, the Rsq, Beta, and Eta values along with some other variables will be written. The text file is in csv format without a variable names header. The contents are the same as the GRAPHDS contents.

FILEMODE specifies whether the TEXTFILE is overwritten or new results are appended. If appending, the structure, particularly the presence of an ID variable, must be the same as the previous contents.
